Inventory Management System (IMS)
Project Overview

The Inventory Management System (IMS) is a Java-based web application designed to manage inventory operations in an organized and efficient manner. The system supports item requesting, approval, issuing, and user management using role-based access control.

This project is developed as a group academic assignment and demonstrates the use of Java Servlets, JSP, MVC Tomcat server, and database integration.

Project Objectives

>> Automate inventory request and approval processes
>>Reduce manual work and errors
>>Manage inventory items efficiently
>>Enforce secure, role-based access control

User Roles and Responsibilities
  Instructor
      Request inventory items
      View request status
  Department Head
      Approve or reject inventory requests
  Storekeeper
       Issue approved inventory items
       Add new inventory items
       View issued items
  Admin
       Manage system users and roles

System Workflow

>>User logs in to the system
>>Instructor submits an item request
>>Department Head reviews and approves or rejects the request
>>storekeeper issues approved items
>>System updates inventory and request records

Technologies Used
  Java (Servlets and JSP)
  HTML and CSS
  MySQL Database
  Apache Tomcat 10
  IntelliJ IDEA
  Git and GitHub

Security Features
  Session-based authentication
  Role-based authorization
  Page access control using filters
  Redirection for unauthorized access

How to Run the Project

Clone the repository:
  git clone https://github.com/selam-mbk1/inventory-management-system.git


Open the project in IntelliJ IDEA
Configure Apache Tomcat 10
Create and configure the MySQL database
Run the application
Access the system using:

http://localhost:8080/IMS
